# Morzelingo 🧩🔊

**Morzelingo** — это геймифицированное веб-приложение для изучения азбуки Морзе, вдохновлённое дизайном и механиками Duolingo. Превратите изучение точек и тире в увлекательную игру с прогрессом, соревнованиями и ежедневной мотивацией.

## ✨ Основные возможности

*   **Стрики (Streaks)**: Ежедневная мотивация для регулярных занятий. Не пропускайте ни дня, чтобы сохранить свою серию!
*   **Уровни и XP**: Зарабатывайте очки опыта (XP) за прохождение уроков и практику. Повышайте свой уровень и открывайте новый контент.
*   **Друзья**: Добавляйте друзей, чтобы следить за их прогрессом и соревноваться в количестве XP.
*   **Структурированные уроки**: Проходите последовательные уроки, постепенно осваивая новые буквы и символы.
*   **Отдельная практика букв**: Тренируйте конкретные буквы, которые у вас вызывают затруднения, без необходимости проходить весь урок заново.

## 🚀 Быстрый старт (Запуск проекта)

Проект написан на Go и не требует сложной настройки окружения, кроме самого компилятора Go.

### Предварительные требования
*   Установленный **Go**.
*   Git (для клонирования репозитория).

### Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/Cautro/morzelingo.git
    ```
2.  **Перейдите в директорию проекта:**
    ```bash
    cd morzelingo
    ```
3.  **Скачайте необходимые зависимости:**
    Проект использует Go модули. Все зависимости будут автоматически загружены при сборке или запуске. Вы можете выполнить команду для синхронизации:
    ```bash
    go mod download
    ```
4.  **Запустите сервер:**
    Главная точка входа находится в `cmd/server`. Для запуска выполните:
    ```bash
    go run ./cmd/server
    ```
    После успешного запуска сервер будет доступен по адресу `http://localhost:8080`

## 🛠 Технологии

*   **Backend**: Go (Golang)
*   **База данных** JSON (Находятся в папке `/data`)

## 📁 Структура проекта

*   `cmd/` - Точки входа в приложение.
*   `server/` - Основной сервер приложения.
*   `internal/` - Приватный код приложения (бизнес-логика, обработчики, модели).
*   `data/` - Папка с джейсонами(БД для этого проекта)
*   `go.mod` - Файл зависимостей Go модуля.
*   `main.go` - старый неоптимизированный стартовый файл (лучше сразу удалить, он тут нужен ведь проект пока проходит "разделение" общего файла на более маленькие и ор необходим чтобы нечего не потерять)
